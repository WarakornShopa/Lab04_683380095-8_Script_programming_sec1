Lab 4.1: Student Management System (List Operations)
วัตถุประสงค์: ให้นักเรียนสร้างโปรแกรมง่ายๆ เพื่อจัดการรายชื่อนักเรียน เพื่อฝึกฝนและทำความเข้าใจเกี่ยวกับการสั่งงาน List (รายการข้อมูล) ในรูปแบบต่างๆ 
สิ่งที่ต้องทำ
1. สร้าง List ว่างเปล่าขึ้นมาตัวหนึ่ง ตั้งชื่อว่า student_names
 student_names = []
2. เพิ่มรายชื่อนักเรียน (Add Students): รับข้อมูลจากผู้ใช้ โดยให้กรอกชื่อนักเรียน 3-5 คน แล้วใช้คำสั่ง append() เพื่อเพิ่มแต่ละชื่อเข้าไปใน List จากนั้นให้พิมพ์ (print) รายชื่อปัจจุบันออกมาดู 
ตัวอย่าง :
student_names.append(“Aphelios”)
student_names.append(“Bard”)
3. ค้นหารายชื่อนักเรียน (Find a Student): ถามผู้ใช้ว่าต้องการค้นหาชื่อใคร จากนั้นใช้เงื่อนไข if item in list: เพื่อตรวจสอบว่ามีชื่อนักเรียนคนนั้นอยู่จริงไหม หากเจอ ให้พิมพ์ตำแหน่งพิกัด (index) ของนักเรียนคนนั้นออกมาโดยใช้คำสั่ง index()
4. ลบรายชื่อนักเรียน (Remove a Student): ถามผู้ใช้ว่าต้องการลบชื่อใครออกจากระบบ ให้ใช้คำสั่ง remove() ในการลบข้อมูลออก โดยต้องดักทางไว้ด้วยในกรณีที่ไม่พบชื่อดังกล่าว (เช่น ใช้ if item in list: ตรวจสอบก่อนสั่ง remove()) จากนั้นให้พิมพ์รายชื่อที่อัปเดตแล้วออกมาดู
5. จัดเรียงรายชื่อ (Sort Students): ทำการเรียงลำดับรายชื่อใน student_names ตามตัวอักษร (ก-ฮ หรือ A-Z) โดยใช้คำสั่ง sort() แล้วพิมพ์ผลลัพธ์ออกมา
6. นับจำนวนนักเรียน (Count Students): พิมพ์จำนวนนักเรียนทั้งหมดที่มีอยู่ใน List ออกมาโดยใช้คำสั่ง len()

โจทย์ท้าทาย (Challenge): ลองสร้างระบบเมนูสั่งการแบบง่ายๆ โดยใช้การวนลูป while (จากบทเรียนสัปดาห์ที่ 3) เพื่อให้ผู้ใช้สามารถกดเลือกได้ว่าจะ เพิ่ม, ลบ, ค้นหา, เรียงลำดับ หรือกดออกจากโปรแกรม
เช่น
1.เพิ่ม
2.ลบ
3.ค้นหา
4.เรียงลำดับ
5.ออกจากโปรแกรม

Lab 4.2: Geometric Calculations (Tuples & Immutability)
วัตถุประสงค์: ให้นักเรียนใช้ Tuple ในการแทนค่าพิกัด 2 มิติ (2D points) และคำนวณหาระยะห่างระหว่างจุด เพื่อให้เห็นภาพชัดเจนเกี่ยวกับคุณสมบัติ Immutability (การไม่สามารถเปลี่ยนแปลงค่าได้ของ Tuple) 

1. กำหนดพิกัดจุด: กำหนดจุด 2 จุดในรูปแบบ Tuple ดังนี้: point1 = (3, 4) และ point2 = (6, 8)
เข้าถึงค่าพิกัด: ดึงค่าและพิมพ์พิกัด x และ y ของทั้งสองจุดออกมาแสดงผล
ตัวอย่าง: "Point 1: x = 3, y = 4"
2. ทดลองแก้ไขค่า (เพื่อพิสูจน์เรื่อง Immutability): ลองเขียนโค้ดเพื่อเปลี่ยนพิกัด x ของ point1 (เช่น พิมพ์สั่งว่า point1[0] = 5) สังเกตข้อผิดพลาดที่เกิดขึ้น (TypeError) และอธิบายเหตุผลว่าทำไมถึงแก้ไม่ได้ (หลังจากทดลองเสร็จแล้ว ให้ใส่เครื่องหมาย # ด้านหน้า เพื่อคอมเมนต์บรรทัดนี้ทิ้งไป)
คำนวณระยะห่าง (Calculate Distance):
นำเข้าโมดูลคณิตศาสตร์: import math
ใช้สูตรหาระยะห่างทางเรขาคณิต: (x2​−x1​)2+(y2​−y1​)2​.
คำนวณระยะห่างระหว่าง point1 และ point2 โดยใช้ฟังก์ชัน math.sqrt() และเครื่องหมายยกกำลัง
พิมพ์ผลลัพธ์ระยะห่างที่คำนวณได้ออกมา
โจทย์ท้าทาย (Challenge): ลองเขียนโค้ดให้รับโดยใช้คำสั่ง input แทนที่จะกำหนดค่าในโปรแกรม แล้วนำมาคำนวณหาระยะห่าง อย่าลืมแปลงชนิดข้อมูล (Type Conversion) ที่ได้มาจากคำสั่ง input() ให้เรียบร้อยก่อนนำไปคำนวณ (เช่น แปลงเป็น integer หรือ float)

Activity 4.1: List vs. Tuple: Scenario Matching
สิ่งที่อยากให้ทำ : สถานการณ์ต่อไปนี้ แต่ละเหตุการณ์ควรใช้ list หรือ tuple เพราะอะไร 
1.ข้อมูลอุณหภูมิประจำวันที่มีการบันทึกไว้ตลอดหนึ่งสัปดาห์ 
2.ค่าสี RGB ของเฉดสีเฉพาะเฉดหนึ่ง (เช่น สีแดงคือ (255, 0, 0))
3.รายการสินค้าในรถเข็นชอปปิงที่สามารถกดเพิ่มหรือลบออกได้ 
4.รายชื่อวันในหนึ่งสัปดาห์เรียงตามลำดับ (จันทร์ - อาทิตย์)
5.คะแนนเก็บของนักเรียนจากการส่งงานชุดต่างๆ ซึ่งอาจมีการอัปเดตหรือแก้ไขคะแนนในภายหลังได้
6.พิกัด GPS ของสถานที่สำคัญหรือจุดแลนด์มาร์กที่ตั้งอยู่กับที่


Activity 4.2: Interactive To-Do List (using List Operations)
สิ่งที่อยากให้ทำ : ให้นักเรียนพัฒนาโปรแกรมจัดการรายการสิ่งที่ต้องทำ (To-Do List) แบบข้อความ (Text-based) อย่างง่าย โดยประยุกต์ใช้ความรู้เรื่องการจัดการ List ที่ได้เรียนมา
สิ่งที่ต้องมี
1. สร้าง List ว่างเริ่มต้นไว้ตัวหนึ่ง ชื่อว่า todo_list
2. ใช้การวนลูป while เพื่อแสดงเมนูคำสั่งให้ผู้ใช้เลือก ดังนี้:
2.1 เพิ่มรายการงาน (Add a task)
2.2 ดูรายการงานทั้งหมด (View tasks)
2.3 ทำเครื่องหมายว่างานเสร็จสิ้นแล้ว (ลบงานออก โดยเลือกจากตำแหน่ง Index หรือตามชื่อคำสั่ง)
2.4 ออกจากโปรแกรม (Exit)
3. ใช้คำสั่ง input() เพื่อรับค่าเมนูที่ผู้ใช้เลือก
4. ใช้คำสั่งจัดการ List ต่างๆ (เช่น append(), remove() หรือ pop(), len()) ในการบริหารจัดการข้อมูลใน todo_list
